# discord-ai-runner: Mermaid画像化送信機能を追加する

- 作成日: 2026-03-02
- Category: タスク
- 実行対象: `projects/discord-ai-runner`（未配置の場合は実リポジトリ配置先に合わせる）

## 概要
Discordは標準でMermaidブロックを図としてプレビュー表示できないため、`discord-ai-runner` 側で Mermaid 記法を画像化し、返信に添付する機能を追加する。

## 背景
- ` ```mermaid ` はDiscord上でコードブロックとして表示される。
- 図として共有するには、Bot側でレンダリングして画像添付する必要がある。

## 要件
1. 受信メッセージ（またはAI応答）から ` ```mermaid ... ``` ` ブロックを抽出できること。
2. Mermaid文字列を `png` または `svg` に変換できること。
3. 変換した画像をDiscord返信に添付できること。
4. Mermaid構文エラー時に、失敗理由をユーザーに返せること。
5. 一時ファイルのクリーンアップを行い、リークしないこと。

## 実装方針（案）
- レンダラ: `@mermaid-js/mermaid-cli` を利用。
- 処理フロー:
  1. Mermaidブロック検出
  2. 一時ファイルへ書き出し
  3. CLIで画像生成
  4. 添付ファイルとして返信
  5. 一時ファイル削除
- 依存追加時は実行環境（Docker/CI/本番）で必要なバイナリ・フォント要件を確認する。

## 受け入れ条件
- 正常系:
  - シーケンス図・フローチャートなど複数種類のMermaid入力で画像添付返信される。
- 異常系:
  - 構文エラー時にエラーメッセージ返信され、Botが落ちない。
  - 空ブロックや極端に大きい入力で安全に失敗処理される。
- 運用:
  - ログにレンダリング失敗理由が残る。
  - 一時ファイルが処理後に残らない。

## 運用メモ（Issue管理）
- 現時点の前提:
  - `ai-workspace` は会社用と私物用でリポジトリが異なる。
  - 現在はローカル `issues/` 管理で大きな問題はない。
- 検討方針:
  - Issue正本をリモート化する場合、会社用と私物用でリモートリポジトリを分離する。
  - `config/settings.yaml.example` をコミットし、実体 `config/settings.yaml` は `cp` で生成して環境ごとのURLを設定する。

  - `agent-sync.sh` は profile を読んで同期先を切替し、誤接続（corp環境でpersonal URLなど）はエラーで止める。
- 将来案:
